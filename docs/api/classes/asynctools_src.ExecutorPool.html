    Class ExecutorPool

    Manages concurrent execution of tools with per-tool concurrency limits, automatic retry with exponential backoff, and timeout via AbortController.

    Each tool gets its own execution queue, bounded by max_concurrent from the ToolDefinition. Excess invocations are queued and processed FIFO.

    Example

    const registry = new ToolRegistry()
    registry.register({ tool_id: 'search', executor: { type: 'http', url: '...' }, max_concurrent: 3 })

    const pool = new ExecutorPool(registry)
    const result = await pool.execute('search', 'call_1', { query: 'nike' })
    console.log(result.status) // 'completed'
