## Unreleased - 2026-03-30

### Changed

- Human role compatibility now accepts generic aliases (`user`, `agent`, `operator`, `supervisor`) alongside the original retail names, and `HumanProfile` accepts `org_id` in addition to `store_id`.
- `StaffAgent` default roles now cover both alias families, and activation utterances such as `Fitaly, aplica...` can execute an inline operational command when the phrase clearly looks actionable.
- Governance, human-role, and hardening docs were aligned with current runtime behavior, including `bus:AGENT_ERROR` and the real `HALF_OPEN` semantics.

### Fixed

- `CircuitBreaker` now allows only one concurrent probe in `HALF_OPEN`.
- `InMemoryBus.publish()` now keeps dispatching after a handler failure, still awaits async handlers, and no longer defers synchronous subscribers to a later microtask.
- `StreamAgent` now emits `bus:AGENT_ERROR` when `onEvent()` fails instead of swallowing the failure silently.
- TTL expiry in `DraftStore` once again publishes `bus:DRAFT_CANCELLED` deterministically on `InMemoryBus`.
- Multi-agent staff handoff flows now surface `STAFF_COMMAND` updates correctly in the UI path.

## v1.2.0 — 2026-03-06

Major v2 rewrite. All new components are available in `fitalyagents` unless noted otherwise.

### License

Changed from Apache 2.0 + Commons Clause to **AGPL-3.0**. Source-available, open source recognized. Commercial closed-source deployments require a separate commercial license — contact the maintainers.

### Breaking changes

- **`NexusAgent` removed** — replaced by `StreamAgent`. Extend `StreamAgent` and implement `onEvent(channel, payload)`.
- **`CapabilityRouter`, `SimpleRouter`, `AgentRegistry`, `LockManager`, `TaskQueue` removed** — the LLM-based dispatcher replaces all static routing.
- **`DispatcherBootstrapper`, `LLMFallbackAgent` removed from dispatcher** — bootstrap now lives in the `IntentTeacher` + `IntentScoreStore` pipeline.

### New — Safety Module

**`SafetyGuard`** — evaluates tool actions against speaker role and approval limits.

**`DraftStore`** (in-memory + Redis) — multi-turn order modification with TTL:

```typescript
const draftId = await draftStore.create(sessionId, orderInput)
await draftStore.update(draftId, { color: 'blue' })
await draftStore.confirm(draftId)
```

**`ApprovalOrchestrator`** — multi-channel human approval (`parallel` or `sequential`):

- `VoiceApprovalChannel` — waits for `bus:SPEECH_FINAL` from the designated approver
- `WebhookApprovalChannel` — posts to a URL, waits for `bus:APPROVAL_WEBHOOK_RESPONSE`
- `ExternalToolChannel` — HTTP callback pattern

### New — Dispatcher v2

**`SpeculativeCache`** — pre-executes SAFE tools during `bus:SPEECH_PARTIAL`. When the final utterance arrives, the tool result is already cached (0ms wait). Confidence threshold: 0.90, margin: 0.15.

**`IntentTeacher`** — learns from corrections at runtime via `addExample()`. Updates the live classifier without restart.

**`IntentScoreStore`** — tracks per-intent hit rate via EMA (α=0.1). `isProduction(intentId)` returns `true` when score ≥ 0.70.

### New — Core Agents

**`StreamAgent`** — abstract base class replacing `NexusAgent`. Handles subscribe/unsubscribe lifecycle, heartbeat, and `onEvent()` dispatch.

**`InteractionAgent`** — streaming LLM agent with full safety pipeline:

- Streams TTS token-by-token via `ttsCallback`
- Routes tool calls through `SafetyGuard` (safe/staged/protected/restricted)
- Integrates speculative cache, `ApprovalOrchestrator`, and `IntentTeacher`

**`ContextBuilderAgent`** — maintains per-session `ConversationContext` (history, ambient, pending draft, action history). Call `getEnrichedContext(sessionId)` before each LLM turn.

**`ProactiveAgent`** — emits `bus:PROACTIVE_TRIGGER` for idle customer, out-of-stock, and draft expiry scenarios.

### New — FitalyVoice (sprints 5.1–5.3)

**`TargetGroupBridge`** — maps hardware speaker events to sessions with priority group management. Publishes `bus:TARGET_GROUP_CHANGED` on every FSM transition.

```typescript
const bridge = new TargetGroupBridge({ bus, sessionManager, storeId: 'kiosk-1' })
await bridge.start()
```

**Ambient Context Pipeline** — `ContextBuilderAgent` extracts `last_product_mentioned` from overheard speech for coreference resolution across turns.

**SPEECH_PARTIAL speculative execution** — `NodeDispatcher` pre-executes SAFE tools during partial speech when confidence ≥ 0.90 and margin ≥ 0.15.

### New — Observability (sprint 6.1)

**`ITracer` / `NoopTracer` / `LangfuseTracer`** — injectable tracing, zero overhead by default:

```typescript
const tracer = new LangfuseTracer(new Langfuse({ publicKey, secretKey }))
const dispatcher = new NodeDispatcher({ bus, classifier, intentLibrary, tracer })
```

`LangfuseTracer` is a duck-type adapter — no `langfuse` peer dependency in core.

### New — Hardening (sprint 6.3, `@fitalyagents/asynctools`)

**`RateLimiter`** — sliding 1-second window enforced per tool via `rate_limit: { requests_per_second }`.

**`CircuitBreaker`** — CLOSED → OPEN → HALF_OPEN FSM with `onOpen`/`onClose` callbacks:

```typescript
registry.register({
  tool_id: 'payment',
  circuit_breaker: { failure_threshold: 5, reset_timeout_ms: 30_000 },
})
const pool = new ExecutorPool(registry, { onOpen: (toolId) => alerting.send(`${toolId} down`) })
```

New `ToolResult.status` values: `'rate_limited'`, `'circuit_open'`.

## v1.1.0 — 2026-02-24

### License

Changed from MIT to **Apache 2.0 + Commons Clause** (superseded by AGPL-3.0 in v1.2.0).

### New — DX utilities (core)

**`SimpleRouter`**
Subscribes to `bus:TASK_AVAILABLE` and routes to `queue:<agent-id>:inbox` automatically. Replaces the manually-written `createSimpleRouter()` pattern that every project had to re-implement.

```typescript
const router = new SimpleRouter({
  bus,
  routes: { product_search: 'work-agent', order_create: 'order-agent' },
  alwaysNotify: ['interaction-agent'],
})
router.start()
```

**`AgentBundle`**
Lifecycle manager for multiple agents and disposables.

```typescript
const bundle = new AgentBundle({ agents: [ia, wa, oa], disposables: [audioQueue] })
await bundle.start()
await bundle.shutdown()
bundle.dispose()
```

### New — LLM enhancements (dispatcher)

**`LLMProvider` interface** — `complete(system, user): Promise<string>` — wraps any LLM backend.

**`ClaudeLLMProvider`** — wraps `@anthropic-ai/sdk`, defaults to `claude-haiku-4-5-20251001`.

**`DispatcherBootstrapper`** — generates intent training examples from agent manifests using an LLM. Idempotent: enriches existing intents, never overwrites.

```typescript
const bootstrapper = new DispatcherBootstrapper({ intentLibrary, llm: new ClaudeLLMProvider() })
await bootstrapper.bootstrapFromManifests([agentManifest])
```

**`LLMDirectClassifier`** — drop-in `IEmbeddingClassifier` replacement. No embeddings; classifies via LLM directly.

```typescript
const classifier = new LLMDirectClassifier({ llm: new ClaudeLLMProvider(), intentLibrary })
await classifier.init()
```

## v1.0.0 — 2026-02-24

Initial public release. Full agent framework with:

### Core (`fitalyagents`)

- **`InMemoryBus` / `RedisBus`** — event bus with pub/sub and Redis-style queues
- **`NexusAgent`** — base class for all agents; `process()` lifecycle
- **`AgentRegistry`** — register and query agents by scope, capability, domain
- **`InMemoryContextStore`** — session-scoped key/value store with access control
- **`InMemoryLockManager`** — distributed-style locks with TTL
- **`InMemorySessionManager`** — full session lifecycle; priority groups; pause/resume
- **`InMemoryTaskQueue`** — task submission, claim, priority ordering
- **`CapabilityRouter`** — routes tasks by capability match
- **`InMemoryAudioQueueService`** — priority audio queue with barge-in support
- **`InMemoryApprovalQueue`** — human-in-the-loop with timeout and webhook support

### Async Tools (`@fitalyagents/asynctools`)

- **`ToolRegistry`** — type-safe tool registration with Zod validation
- **`ExecutorPool`** — parallel tool execution with injection strategies
- **`AsyncAgent`** — wraps any LLM to add async parallel tool execution
- Execution modes: `sync`, `async`, `fire_forget`, `deferred`
- Injection strategies: `inject_when_all`, `inject_when_ready`, `inject_on_timeout`

### Dispatcher (`fitalyagents/dispatcher`)

- **`NodeDispatcher`** — classifies speech events and publishes `TASK_AVAILABLE`
- **`InMemoryEmbeddingClassifier`** — embedding-based cosine similarity classifier
- **`InMemoryLLMFallbackAgent`** — LLM-powered fallback for low-confidence classifications
- **`InMemoryIntentLibrary`** — intent definitions and example storage

### Voice Retail example

- `InteractionAgent`, `WorkAgent`, `OrderAgent` — full production-grade agents
- `createApprovalWebhookHandler` — Express/Hono webhook for approval decisions
- 73 E2E tests covering full pipeline, order lifecycle, multi-session, cancel chains

### Documentation

- 5 guides: getting-started, asynctools, add-new-agent, training-the-dispatcher, rust-dispatcher
- TypeDoc API reference (`pnpm typedoc`)
