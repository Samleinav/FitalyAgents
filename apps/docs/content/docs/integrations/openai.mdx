The `openai` package is the most direct way to wire a model into the current FitalyAgents runtime. The main adapter point is `IStreamingLLM`, which powers `InteractionAgent`.

Runnable example: `examples/openai-agent`

## Install

```bash
pnpm add openai
```

Set your API key:

```bash
OPENAI_API_KEY=...
```

## Adapter shape

`InteractionAgent` expects an object that implements:

```ts
interface IStreamingLLM {
  stream(params: {
    system: string
    messages: Array<{ role: 'user' | 'assistant' | 'tool'; content: string; tool_call_id?: string }>
    tools?: Array<{ name: string; description?: string; input_schema?: unknown }>
  }): AsyncIterable<
    | { type: 'text'; text: string }
    | { type: 'tool_call'; id: string; name: string; input: unknown }
    | { type: 'end'; stop_reason: 'end_turn' | 'tool_use' | 'max_tokens' }
  >
}
```

With OpenAI, a minimal adapter looks like this:

```ts
import OpenAI from 'openai'
import type { IStreamingLLM, LLMStreamChunk } from 'fitalyagents'

class OpenAIStreamingLLM implements IStreamingLLM {
  private client = new OpenAI({ apiKey: process.env.OPENAI_API_KEY })

  async *stream(params: {
    system: string
    messages: Array<{ role: 'user' | 'assistant' | 'tool'; content: string; tool_call_id?: string }>
    tools?: Array<{ name: string; description?: string; input_schema?: unknown }>
  }): AsyncIterable<LLMStreamChunk> {
    const response = await this.client.chat.completions.create({
      model: process.env.OPENAI_MODEL ?? 'gpt-4o-mini',
      temperature: 0.2,
      messages: [
        { role: 'system', content: params.system },
        ...params.messages.map((message) =>
          message.role === 'tool'
            ? {
                role: 'tool' as const,
                tool_call_id: message.tool_call_id ?? 'tool_call_missing',
                content: message.content,
              }
            : {
                role: message.role,
                content: message.content,
              },
        ),
      ],
    })

    const message = response.choices[0]?.message
    const text = typeof message?.content === 'string' ? message.content : ''
    if (text) {
      yield { type: 'text', text }
    }

    for (const toolCall of message?.tool_calls ?? []) {
      yield {
        type: 'tool_call',
        id: toolCall.id,
        name: toolCall.function.name,
        input: JSON.parse(toolCall.function.arguments || '{}'),
      }
    }

    yield {
      type: 'end',
      stop_reason: (message?.tool_calls?.length ?? 0) > 0 ? 'tool_use' : 'end_turn',
    }
  }
}
```

## Using it with `InteractionAgent`

```ts
import { InMemoryBus, InMemoryContextStore, InteractionAgent, SafetyGuard } from 'fitalyagents'

const agent = new InteractionAgent({
  bus: new InMemoryBus(),
  llm: new OpenAIStreamingLLM(),
  contextStore: new InMemoryContextStore(),
  toolRegistry: new Map(),
  executor: {
    async execute(toolId, input) {
      return { toolId, input }
    },
  },
  safetyGuard: new SafetyGuard({ toolConfigs: [] }),
})
```

## Tool calling

If you want OpenAI tool calls to flow through FitalyAgents governance, pass `tools` through your adapter and emit `tool_call` chunks. The runnable example in `examples/openai-agent` already does this. `InteractionAgent` will route them through:

- `SafetyGuard`
- `InMemoryDraftStore`
- `ApprovalOrchestrator`

For a ready-to-run baseline, start from `examples/openai-agent` and then add tools to the `toolRegistry`.
