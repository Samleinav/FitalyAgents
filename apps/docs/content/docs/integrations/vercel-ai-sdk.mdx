The Vercel AI SDK is a good fit when you want a provider-agnostic model layer with straightforward streaming primitives.

Runnable example: `examples/vercel-ai-sdk-agent`

## Install

```bash
pnpm add ai @ai-sdk/openai
```

Set your API key:

```bash
OPENAI_API_KEY=...
```

## Adapter shape

The most direct integration is adapting `streamText()` to `IStreamingLLM`.

The runnable example is intentionally text-first. `InteractionAgent` history currently keeps `tool_call_id`, but not the richer tool metadata that AI SDK `ModelMessage` tool parts expect, so the baseline adapter folds prior tool outputs back into assistant text. If you need full AI SDK tool loops, extend the message mapper with your own tool metadata.

```ts
import { streamText, type ModelMessage } from 'ai'
import { openai } from '@ai-sdk/openai'
import type { IStreamingLLM, LLMStreamChunk } from 'fitalyagents'

function toModelMessages(
  messages: Array<{ role: 'user' | 'assistant' | 'tool'; content: string; tool_call_id?: string }>,
): ModelMessage[] {
  return messages.map((message) => {
    if (message.role === 'tool') {
      return {
        role: 'assistant',
        content: `[Tool result ${message.tool_call_id ?? 'unknown'}]\n${message.content}`,
      }
    }

    return {
      role: message.role,
      content: message.content,
    }
  })
}

class VercelAIStreamingLLM implements IStreamingLLM {
  private model = openai(process.env.VERCEL_AI_MODEL ?? 'gpt-4o-mini')

  async *stream(params: {
    system: string
    messages: Array<{ role: 'user' | 'assistant' | 'tool'; content: string; tool_call_id?: string }>
    tools?: Array<{ name: string; description?: string; input_schema?: unknown }>
  }): AsyncIterable<LLMStreamChunk> {
    const result = streamText({
      model: this.model,
      system: params.system,
      messages: toModelMessages(params.messages),
    })

    for await (const chunk of result.textStream) {
      if (chunk) {
        yield { type: 'text', text: chunk }
      }
    }

    yield { type: 'end', stop_reason: 'end_turn' }
  }
}
```

## Using it with `InteractionAgent`

```ts
import { InMemoryBus, InMemoryContextStore, InteractionAgent, SafetyGuard } from 'fitalyagents'

const agent = new InteractionAgent({
  bus: new InMemoryBus(),
  llm: new VercelAIStreamingLLM(),
  contextStore: new InMemoryContextStore(),
  toolRegistry: new Map(),
  executor: {
    async execute(toolId, input) {
      return { toolId, input }
    },
  },
  safetyGuard: new SafetyGuard({ toolConfigs: [] }),
})
```

## When to use it

Use the Vercel AI SDK integration when:

- you want to swap providers without rewriting your agent shell
- you care about streaming UX
- you want the model layer to stay thin and keep orchestration in FitalyAgents

The example in `examples/vercel-ai-sdk-agent` is intentionally small so you can extend it with tools or governance later.
